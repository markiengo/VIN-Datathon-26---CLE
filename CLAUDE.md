## Part 2 — Analytics / Visualization (60pts — highest weight)

This is the most important section. Judges score on four levels:
- **Descriptive → Diagnostic → Predictive → Prescriptive**
- Consistently reaching Prescriptive across multiple analyses = max score.

Rules:
- Every chart must have: title, axis labels, annotations with specific numbers.
- Do not produce obvious/generic analyses (e.g., "revenue increased over time" alone is not enough).
- Connect multiple tables — cross-table insight signals creativity.
- Pair each chart with: what it shows + key numbers + business recommendation.
- Target 3–5 deep analyses over 10 shallow ones.

<important if="writing Part 2 analysis text">
Business recommendations must be specific and quantified, not generic.
BAD: "the company should improve its return process"
GOOD: "Streetwear returns (wrong_size: 42%) suggest investing in a virtual fit tool — targeting even a 10% reduction in wrong_size returns would recover ~X VND/month in refund costs"
</important>

## Part 3 — Sales Forecasting (Kaggle submission)

### Problem
Predict `Revenue` for each date in sample_submission.csv (2023-01-01 → 2024-07-01).

### Submission file rules (CRITICAL — disqualification risk)
- File must be named `submission.csv`
- Columns: `Date, Revenue, COGS`
- Row ORDER must exactly match `sample_submission.csv` — do NOT sort or shuffle
- Do NOT use Revenue or COGS from the test period as model features (data leakage → disqualification)
- Do NOT use external data outside the provided dataset files
- Set random seeds for all stochastic operations: `random.seed(42)`, `np.random.seed(42)`, model `random_state=42`

### Required in the technical report
- Feature importances OR SHAP values OR partial dependence plots
- Explanation in business language (not just variable names)
- Time-aware cross-validation (no shuffled K-fold — use TimeSeriesSplit or walk-forward)
- Leakage control: no future information in features at any point in the pipeline

### Feature engineering from available data (no external data allowed)
Useful signal sources to derive features from:
- `sales.csv`: lag features (7, 14, 30d), rolling means, rolling std, month/dow/quarter
- `web_traffic.csv`: lagged sessions, bounce_rate, traffic_source mix — starts 2013, so align carefully
- `inventory.csv`: aggregate stockout_flag, fill_rate monthly → join on date
- `orders.csv` / `payments.csv`: aggregate daily order counts, avg payment value
- `promotions.csv`: flag whether a promotion was active on each date (join on date range)
- Calendar: Vietnamese holidays, Tết (Lunar New Year), mid-year sale seasons
